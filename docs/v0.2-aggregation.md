## 🚀 Click-Rem v0.2 — Aggregation Layer

v0.2 introduces a **type-safe aggregation and filtering system** on top of ClickHouse.

This version focuses on:

* metrics (sum, avg, min, max, count)
* dimensions (grouping)
* filters (typed)
* time helpers (query DSL)

---

## 🧠 Aggregation API

```ts
await db.aggregate(table, {
  metrics: { ... },
  dimensions?: { ... },
  where?: { ... }
})
```

---

## 📊 Metrics

Supported metrics:

```ts
sum(column)
avg(column)
min(column)
max(column)
count()
```

---

### Example

```ts
metrics: {
  total: sum(events.amount),
  avgAmount: avg(events.amount),
  minAmount: min(events.amount),
  maxAmount: max(events.amount),
  totalEvents: count()
}
```

---

### Type Safety

* `sum / avg / min / max` → only numeric columns allowed
* `count` → no column required

---

## 🧩 Dimensions

Used for grouping:

```ts
dimensions: {
  date: day(events.timestamp)
}
```

---

### Current Support

* `day(column)` → groups by date

---

## 🔍 Filters (Typed)

```ts
where: {
  event: 'purchase',
  amount: { gt: 50 },
  timestamp: { gte: new Date() }
}
```

---

### Supported Operators

```ts
eq
gt
gte
lt
lte
in
```

---

## ⏱️ Time Helpers

Time helpers provide a **clean DSL for date filtering**.

---

### Basic Helpers

```ts
last7Days()
last30Days()
thisMonth()
thisWeek()
```

---

### Flexible Helpers

```ts
last(14, 'days')
between(start, end)
before(date)
after(date)
range({ from, to })
```

---

### Example

```ts
where: {
  timestamp: last7Days()
}
```

---

## 🧠 Result Typing

Results are fully typed:

```ts
const result = await db.aggregate(...)

// result shape:
{
  date: string
  total: number
  average: number
}[]
```

---

## 🧠 Design Principles

* schema-first approach
* type-safe analytics
* minimal abstraction over ClickHouse
* composable query building

---

## ⚠️ Limitations (v0.2)

* only `MergeTree` engine supported
* no joins
* no pre-aggregations
* limited dimension types
* schema is not updated after `ALTER`
