## 🧱 Click-Rem v0.1 — Schema & DDL

Click-Rem v0.1 provides a **type-safe schema layer and DDL utilities for ClickHouse**.

This version focuses on:

* defining tables with TypeScript
* creating and managing tables
* inserting data safely
* executing raw SQL when needed

---

## 📦 Schema Definition

Tables are defined using `chTable`:

```ts
const events = chTable(
  'events',
  {
    id: ch.uuid(),
    event: ch.string(),
    amount: ch.float64(),
    timestamp: ch.datetime()
  },
  {
    orderBy: 'timestamp'
  }
)
```

---

### Config Options

```ts
type TableConfig<T> = {
  orderBy: keyof T | (keyof T)[]
  primaryKey?: keyof T | (keyof T)[]
  partitionBy?: string
}
```

---

### Notes

* `orderBy` → defines primary index (required)
* `primaryKey` → optional optimization
* `partitionBy` → optional partitioning expression
* fully type-safe (`keyof T`)

---

## ⚙️ Client

Create a client using:

```ts
const db = createClient({
  url: string,
  username?: string,
  password?: string,
  database?: string
})
```

---

## 🧩 Available Methods

---

### 🧱 `createTable`

```ts
await db.createTable(table)
```

* creates table using schema
* uses `MergeTree` engine
* fails if schema invalid

---

### 🔁 `ensureTable`

```ts
await db.ensureTable(table)
```

* safe version of create
* uses `CREATE TABLE IF NOT EXISTS`
* idempotent

---

### ➕ `insert`

```ts
await db.insert(table, data)
```

* accepts:

  * single object
  * array of objects

```ts
await db.insert(events, {
  id: '...',
  event: 'signup',
  amount: 10,
  timestamp: new Date()
})
```

* fully type-safe based on schema

---

### 🛠️ `addColumn`

```ts
await db.addColumn(table, columnName, column)
```

```ts
await db.addColumn(events, 'country', ch.string())
```

* adds new column to table
* does not update TypeScript schema (v0.1 limitation)

---

### 🗑️ `dropColumn`

```ts
await db.dropColumn(table, columnName)
```

```ts
await db.dropColumn(events, 'amount')
```

---

### 🧹 `truncateTable`

```ts
await db.truncateTable(table)
```

* removes all data
* keeps structure

---

### 🗑️ `dropTable`

```ts
await db.dropTable(table)
```

* deletes table completely

---

### 🧾 `sql` (raw queries)

```ts
await db.sql`SELECT * FROM table`
```

* tagged template function
* returns typed results:

```ts
const rows = await db.sql<{ count: number }>`
  SELECT count() as count FROM events
`
```

---

### 🔌 `close`

```ts
await db.close()
```

* closes ClickHouse connection

---

## 🧠 Design Principles

* schema-first approach
* type-safe operations
* minimal abstraction over ClickHouse
* raw SQL always available

---

## ⚠️ Limitations (v0.1)

* schema is not updated after `addColumn`
* only `MergeTree` engine supported
* no aggregation helpers
* no query builder
* no reporting abstraction

---

## 🚀 Next (v0.2)

* metrics system
* reporting layer
